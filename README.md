# OpenCode 双窗口自动对话

两个opencode实例通过命名管道自动循环对话。用户抛出话题后，两个AI会围绕话题自动展开讨论。

## 快速开始

### 1. 启动对话
```bash
./run.sh
```
脚本会提示你输入初始话题：
- 输入自定义话题后按回车（如"人工智能的伦理问题"）
- 直接按回车使用默认话题

### 2. 对话过程
- **窗口A（分析者）**：分析并回应话题
- **窗口B（评论者）**：评论并提问
- 两个窗口自动循环对话，直到你停止

### 3. 停止对话
按 `Ctrl+C` 或运行：
```bash
./stop-chat.sh
```

## 功能特点
- 用户可自定义初始话题
- 两个AI角色：分析者（窗口A）和评论者（窗口B）
- **记忆功能**：AI能记住之前的对话内容，保持连贯性
- 自动循环对话，直到用户停止
- 实时显示彩色对话内容
- 自动记录日志和历史

## 工作原理
1. 用户输入初始话题
2. 窗口A（分析者）发送话题给窗口B（评论者）
3. 窗口B评论并提问，回复给窗口A
4. 窗口A分析并回应，回复给窗口B
5. 循环继续，直到用户按 Ctrl+C 停止

### 记忆功能
- 每次回复都会参考最近的对话历史（约20条消息）
- 历史记录保存在 `/tmp/opencode_chat_history.txt`
- AI能保持对话的连贯性和深度

## 对话示例
```
用户输入：人工智能

A → B: 人工智能
B → A: 人工智能正在深刻重塑人类社会的生产方式与思维模式，但其快速发展也引发了关于伦理边界与人类主体性的深层思考——我们是否已经准备好面对一个机器智能可能超越人类认知边界的未来？
A → B: AI既是工具延伸也是认知挑战，关键不在于"是否准备好"，而在于人类能否主动塑造规则而非被动适应变化。
B → A: 这个观点切中了人机关系的核心张力，但"主动塑造规则"的前提是——谁有资格代表"人类"来制定规则？
...
```

## 配置文件
脚本会自动创建配置文件 `~/.opencode-chat.conf`，可以修改以下配置：

```bash
# 超时时间（秒）
TIMEOUT=30

# 初始消息
INITIAL_MESSAGE="你好，我是实例A，让我们开始对话吧！"

# 日志文件位置
LOG_FILE="/tmp/opencode_chat.log"
```

## 故障排除

### 1. 管道已存在
```bash
rm -f /tmp/opencode_pipe_*
```

### 2. 进程未停止
```bash
./stop-chat.sh
pkill -f "opencode run"
```

### 3. opencode未安装
```bash
curl -fsSL https://opencode.ai/install | bash
```

### 4. 对话卡住
检查日志文件：
```bash
tail -f /tmp/opencode_chat.log
```

## 文件说明
- `chat-memory.sh` - 主脚本（带记忆功能的双窗口自动对话）
- `run.sh` - 启动脚本
- `stop-chat.sh` - 停止脚本
- `install.sh` - 安装脚本
- `README.md` - 说明文档

## 注意事项
- 每次回复都会启动新的opencode会话，可能较慢
- 对话可能因超时而中断
- 建议在测试环境中使用
- 日志文件会不断增长，定期清理
